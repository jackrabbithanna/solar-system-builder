# Alpha Centauri Preset

The built-in `Alpha Centauri` preset lives in `src/presets/binary_system.json`.
The filename and id remain `binary_system.json` and `builtin-binary-system` for
compatibility with preset ordering and saved-system references, but the visible
name is `Alpha Centauri`.

This preset is an approximate 2D simulation seed. It is intended to provide a
recognizable Alpha Centauri-like system for interactive playback, not a
current-date ephemeris.

## Included Bodies

The preset includes the three known stars:

- `Alpha Centauri A`
- `Alpha Centauri B`
- `Proxima Centauri`

It also includes confirmed and candidate planets:

- `Proxima d`, confirmed, parented to Proxima Centauri
- `Proxima b`, confirmed, parented to Proxima Centauri
- `Proxima c Candidate`, disputed/candidate, parented to Proxima Centauri
- `Alpha Centauri A Candidate`, candidate, parented to Alpha Centauri A

Candidate status is represented in the body name and preset description. The
model currently has no dedicated candidate/confirmed field, so all bodies are
simulated the same way.

## Construction

The preset is generated by `scripts/generate_alpha_centauri_preset.py`. That
script is the source of truth for the state-vector math. Run it without
arguments to print the generated JSON, or with `--write` to overwrite
`src/presets/binary_system.json`.

All stored values use SI units:

- mass: kg
- radius and position: meters
- velocity: meters per second
- time settings: seconds

The generator uses:

- Alpha Centauri A mass: `1.0788` solar masses
- Alpha Centauri B mass: `0.9092` solar masses
- Proxima Centauri mass: `0.1221` solar masses
- Alpha Centauri AB semimajor axis: `23.299 AU`
- Alpha Centauri AB eccentricity: `0.52`
- Proxima separation from the AB barycenter: `13,000 AU`
- Proxima approximate semimajor axis around AB: `8,700 AU`

Alpha Centauri A and B are placed in a simplified apastron-like 2D state:

1. Compute AB separation as `a * (1 + e)`.
2. Compute relative speed using vis-viva:
   `sqrt(G * (mA + mB) * (2 / r - 1 / a))`.
3. Split positions and velocities around the AB barycenter according to mass.

Proxima is placed at `13,000 AU` from the AB barycenter with a simple bound-orbit
speed from the same vis-viva relation. A small counter-velocity is applied to the
AB pair so the wide system is not entirely one-sided in momentum.

Planet and candidate positions are circular local orbits around their parent
star. The generator computes each local circular velocity as:

`sqrt(G * parent_mass / orbital_radius)`

Then it adds the parent star's global position and velocity so every child body
has a valid global state vector for the flat N-body physics model.

## Planet Assumptions

The planet data is intentionally compact and approximate:

- Proxima d: `0.029 AU`, `0.26` Earth masses, circular seed
- Proxima b: `0.0485 AU`, `1.27` Earth masses, circular seed
- Proxima c Candidate: `1.49 AU`, `7.0` Earth masses, circular seed
- Alpha Centauri A Candidate: `2.0 AU`, `100` Earth masses, circular seed

The candidate bodies are included because the preset is meant to show the
interesting current Alpha Centauri system, including disputed or tentative
objects. They should not be treated as confirmed discoveries.

## Simulation Settings

The preset stores schema v4 settings:

- `visible_step_s`: `30` days
- `accuracy_profile`: `fast`
- `distance_unit`: `AU`
- `view_mode`: `log_overview`
- `simulation_scope`: `auto`
- `trail_sample_interval_s`: `30` days

`log_overview` is important because Proxima is thousands of AU away from Alpha
Centauri AB while the Proxima planets are much less than 2 AU from their parent.
A linear fit-system view would make the close Proxima planetary system visually
collapse.

`auto` simulation scope keeps the whole-system Alpha Centauri view interactive by
using a stellar overview: only Alpha Centauri A, Alpha Centauri B, and Proxima
Centauri are advanced while child planets are collapsed for display. Switching to
follow-selected or focused subsystem mode advances the selected star and its
children in detail, so Proxima b/d/c Candidate or the Alpha Centauri A Candidate
can show local orbital paths without forcing every whole-system step to use the
shortest planetary timestep.

## Limitations

This preset deliberately does not attempt to model:

- full 3D orbital inclination;
- current-date sky-plane state vectors;
- the exact Alpha Centauri AB orbital phase;
- Proxima's full long-period orbit;
- eccentric planetary orbits;
- uncertainty ranges for radial-velocity minimum masses;
- observational confidence levels beyond candidate naming.

The physics engine still treats all bodies as a flat N-body system. The
`parent_id` values are for display, sorting, distance readouts, and generating
reasonable initial child states; they do not restrict gravitational interaction.

## Validation

Regression coverage verifies that:

- the generated JSON matches the committed preset;
- the preset round-trips through the model schema;
- the preset contains three stars;
- confirmed and candidate planets are present and parented correctly;
- star separations are in the expected AU/kAU ranges;
- stars move under simulation.

Run:

```sh
python3 -m unittest discover -s tests
meson test -C builddir
```
